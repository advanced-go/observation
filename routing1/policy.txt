Routing policy and processing

If a host is configured - static routing, then route to that host only.
  - Ignore versioning

Policy Options -
  1. Static routing
     - Can be used for authority or URL routing.
     - Single host, no failover

  2. Dynamic routing
     - Only authority based routing
     - Policy required with configured templates

  3. Dynamic routing with conversion
       - Only authority based routing
       - Policy required with configured templates
       - Conversion policy required
